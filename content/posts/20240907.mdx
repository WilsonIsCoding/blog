## 大綱
<p>
在許多對外網站的需求清單中，主管往往會特別強調某些關鍵頁面需要進行 <b>SEO（搜尋引擎優化）</b>。在現今各種 SPA（Single Page Application）框架盛行的前端時代，如何兼顧效能與 SEO 已成為開發時的重要課題。
</p>
<p>
以 <b>Next.js</b> 與 <b>Nuxt.js</b> 為代表的框架，已提供相對成熟的 SSR（Server-Side Rendering）與預渲染機制，使 SEO 實作更為穩定與高效。
</p>
<p>
網路上對於 SEO 評分機制已有大量文章探討，因此本篇聚焦於實際案例 —— 在樂居的關鍵頁面中，身為一名前端工程師，我是如何透過技術手段有效提升 SEO 評分與使用體驗。
</p>

## 需求背景
<p>
該頁面具備可分享性，使用者可直接透過網址進入，也能從主網站其他入口導向。這代表頁面需同時支援 CSR（Client-Side Rendering）與 SSR 渲染模式，以兼顧首次載入速度與後續互動效能。
</p>
<p>
此外，頁面資料由多達八支 API 組成，涵蓋建案資訊、廣告模組、社區推薦、街景影像等內容。部分資料參數甚至需跨外網、內網或串接其他 API 回傳結果，資料整合的複雜度相當高。
</p>

## 圖片壓縮（Image Optimization）
<p>
圖片壓縮是提升載入效能的首要步驟。透過壓縮圖片可大幅減少頁面載入時間，同時避免圖片阻塞其他資源的加載。
</p>
<p>
此外，為圖片設定 <code>alt</code>、<code>title</code> 與 <code>srcset</code> 屬性，能協助搜尋引擎理解圖片內容，並在圖片無法顯示時提供替代文字。這不僅強化了 SEO，也提升了整體網站的可用性與無障礙程度。
</p>

## 延遲載入（Lazy Loading）
<p>
在實作上，我們採用滾動觸發式載入（scroll-triggered loading）策略。當使用者滾動至指定位置時，才啟動對應 API 請求。此作法可顯著縮短 FCP（First Contentful Paint）時間，讓使用者更快看到主要內容。
</p>
<p>
藉由延遲非關鍵資源的載入，我們有效降低了初始渲染的阻塞情況，進一步提升頁面效能與使用者體驗。
</p>

## API 快取（API Caching）
<p>
API 快取可在前後端層面同時進行。前端可透過 <code>sessionStorage</code> 儲存部分 API 回應，減少重複請求，降低伺服器負載，並加速後續頁面載入。
</p>
<p>
在實務上，我們曾遇到一個挑戰：如何在不影響 FCP 的前提下，完成所有 API 資料的載入與渲染。因為在部分操作中，使用者會直接滾動至指定區域，若資料尚未載入完成，就可能出現滾動兩次的問題。
</p>
<p>
我們的解法是將 API 呼叫分階段進行：第一次僅請求關鍵資料以確保最快的 FCP，畫面渲染完成後，再依序請求剩餘的非關鍵資料。此方式在不犧牲使用者體驗的前提下，大幅提升了整體載入速度與穩定性。
</p>
